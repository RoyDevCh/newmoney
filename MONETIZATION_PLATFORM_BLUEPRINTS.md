# Monetization Platform Blueprints

## Core Rule

Content is not "good" because it reads smoothly.
Content is good only when it does all four:

1. Stops the scroll
2. Builds trust fast
3. Gives an executable takeaway
4. Pushes one clear conversion action

## Zhihu

Best monetization format:
- red/black list
- buyer's guide
- workflow teardown
- ROI comparison

Structure:
- first paragraph: sharp conclusion
- middle: evidence, scenarios, checklist
- end: comment or save CTA

Must have:
- source/test wording
- practical comparison
- strong "who should buy / who should skip"

Avoid:
- empty macro opinions
- soft lifestyle fluff

## Xiaohongshu

Best monetization format:
- practical checklist
- personal workflow shortcut
- before/after efficiency gap
- small-cost high-return tool list

Structure:
- title: emotional tension + concrete payoff
- body: short blocks, one point per block
- end: save/follow CTA

Must have:
- scene
- low-friction action
- obvious value density

Avoid:
- pretending to be too expert
- long academic explanations

## Douyin

Best monetization format:
- mistake correction
- 3-step script
- one-problem one-solution
- "most people use this wrong"

Structure:
- first 3 seconds: conflict
- middle: short spoken beats
- end: follow/homepage/comment keyword CTA

Must have:
- verbal rhythm
- visualizable beats
- one memorable conclusion

Avoid:
- long clauses
- abstract analysis

## Bilibili

Best monetization format:
- comparison video
- workflow demo
- experiment result
- niche knowledge with proof

Structure:
- hook
- evidence
- segmented takeaways
- closing CTA for triple support or resource comment

Must have:
- proof-first framing
- enough length to feel worth watching
- thumbnail-ready contrast point

Avoid:
- generic motivational tone
- shallow listicles with no test context

## Asset Rule

Every publishable draft should also ship with:

1. one premium cover prompt
2. one negative prompt
3. one composition note
4. one platform aspect ratio

No content pack is complete without the asset layer.
