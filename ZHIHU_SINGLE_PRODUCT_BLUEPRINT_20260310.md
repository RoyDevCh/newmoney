# 知乎单品文章生产蓝图

更新时间：2026-03-10

## 1. 目标

把知乎单品文章从“泛评测长文”升级成更接近搜索流量专栏的内容产品。

核心目标：

1. 首屏先完成购买判断
2. 证据和宣传口径分层
3. 明确适合谁 / 不适合谁
4. 能承接后续型号对比、商品决策和视频母稿复用

## 2. 证据优先级

默认按下面顺序使用证据：

1. 官方产品页 / 规格页 / 发布信息
2. 官方新闻稿 / 发布会文字稿 / 媒体资料包
3. 首批媒体上手 / 公开评测 / 标准化测试
4. 拆解 / 跑分 / 工程向资料
5. 社区讨论 / 首批用户反馈

约束：

1. 社区讨论只能作为问题雷达，不能直接当成硬事实
2. 任何营销话术都必须写成“宣传点 / 当前口径 / 仍待验证”
3. 单品文章严禁伪装成“我自己长期深测”

## 3. 单品文章五种母版

### 3.1 快速结论型「值不值」

适合：

- `XX 值不值得买`
- `XX 现在能买吗`
- `XX 要不要等`

必须包含：

1. 一句话结论
2. 三条最关键理由
3. 适合谁
4. 不适合谁
5. 现在买还是等等

### 3.2 数据验证型「营销打假」

适合：

- `XX 是不是噱头`
- `XX 宣传是不是吹过了`
- `XX 参数党怎么看`

必须包含：

1. 宣传点是什么
2. 当前证据能证实什么
3. 当前证据不能证实什么
4. 这件事真正影响谁
5. 还要继续观察什么

### 3.3 体验分层型「买前模拟」

适合：

- `XX 适合谁`
- `XX 通勤能买吗`
- `XX 学生/女生/家庭用户怎么选`

必须包含：

1. 用户画像分层
2. 场景判断
3. 容易买错的人群
4. 替代选择

### 3.4 工程硬核型「拆解 + 跑分」

适合：

- `XX 散热怎么样`
- `XX 跑分靠谱吗`
- `XX 影像 / AEB / 架构到底行不行`

必须包含：

1. 测试或拆解口径
2. 结构变化
3. 数据意味着什么
4. 哪些结论仍需长期验证

### 3.5 汽车发布专用「上市说明书」

适合：

- `XX 新车值不值得订`
- `XX 上市后怎么买`
- `XX 补能 / 智驾 / 续航怎么看`

必须包含：

1. 上市与版本信息
2. 续航 / 补能 / 智驾口径
3. 适合谁
4. 风险点
5. 现在订还是等等

## 4. 单品文章固定骨架

所有单品文章默认至少包含以下骨架：

1. 首屏结论
2. 证据口径说明
3. 已确认信息
4. 仍待验证信息
5. 适合谁 / 不适合谁
6. 现在买还是等等

## 5. 质检新增标准

单品文章新增两个重点检查项：

1. `single_product_evidence_weak`
   - 证据信号不够，不能上线
2. `marketing_hype_heavy`
   - 营销词密度超过证据密度，不能上线
