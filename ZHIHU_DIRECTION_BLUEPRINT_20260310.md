# 知乎平台内容方向蓝图

更新时间：2026-03-10

## 1. 方向结论

知乎后续主方向固定为：

- 高决策成本科技消费
- 高专业信任的搜索型长文与回答
- 以收藏、搜索承接、后续转化为主，而不是泛阅读量

不再把知乎当作泛科技随笔平台使用。

## 2. 目标受众

目标受众不是“泛科技爱好者”，而是：

- 有明确购买计划的人
- 面对高试错成本的人
- 愿意读长文做判断的人
- 对品牌、型号、预算、使用场景敏感的人
- 后续有较高商品转化潜力的人

## 3. 选题铁律

知乎优先做这些赛道：

- 数码与家电
- 智能家居
- 智能穿戴
- 汽车与出行
- 网络服务与开发者工具

知乎尽量不做这些方向：

- 时政
- 娱乐八卦
- 泛情绪观点
- 低决策成本碎片热点
- 没有明确决策价值的泛科普

## 4. 最优内容形态

知乎优先生产以下形态：

1. 搜索型选购长文
2. 品牌与型号对比文
3. 预算段与场景段决策文
4. 高价值问题回答
5. 长期使用与避坑白皮书

## 5. 成稿必须包含

每篇知乎长文默认必须包含：

1. 首屏直接答案
2. 预算分段
3. 使用场景分层
4. 品牌或型号差异
5. 明确推荐与不推荐
6. 表格、清单或结论块
7. 来源说明、评测口径或公开资料信号
8. 结尾只保留一个动作

## 6. 标题规则

知乎标题优先覆盖以下搜索意图：

- `2026 XX 推荐`
- `XX 怎么选`
- `XX 哪个牌子好`
- `XX 值不值得买`
- `XX 和 XX 怎么选`
- `XX 预算怎么分`

标题要求：

- 必须出现核心品类词
- 最好再带预算、场景、品牌差异中的一项或多项
- 不写成泛观点标题
- 不写成“别再这样做了”这类泛模板标题

## 7. 正文结构模板

标准结构：

1. 先给答案
2. 预算分段
3. 使用场景分层
4. 品牌与型号差异
5. 不推荐情况
6. 结论清单 / 表格
7. 单一转化动作

正文目的不是“讲清一个方法论”，而是“让读者缩小决策范围”。

## 8. 商业化路径

知乎的商业化核心不是流量分成，而是高信任转化：

1. 知乎好物
2. 商品卡 / CPS
3. 清单资料承接
4. 系列专栏沉淀后再做高客单转化

## 9. 与视频的关系

知乎长文可以直接作为哔哩哔哩 / 西瓜视频的母体脚本。

可复用的不是全文照搬，而是：

- 预算段
- 场景段
- 品牌差异段
- 不推荐段
- 结论清单

## 10. 当前系统已落地的代码点

已接入：

- `platform_direction_policy.py`
- `vertical_content_policy.py`
- `autopipeline_brain_content_publisher.py`
- `content_autotune_runner.py`

已生效的行为：

1. 主调度会把知乎方向作为平台级约束传给 main-brain
2. 生成器会显式要求知乎覆盖搜索意图
3. 选购类知乎长文会被强制拉向预算段、场景段、品牌差异、不推荐、结论清单
4. 标签和标题会优先向“选购指南 / 品牌对比 / 预算分段”收拢

## 11. 还没做完

知乎方向已经接入，但还没完全做到稳定 7 成参考专栏水平，当前主要缺口：

1. 还没有稳定的品牌 / 型号知识库
2. 还没有自动生成更强的参数表和价格走势表
3. 还没有把真实产品图自动和正文段落绑定
4. 还没有把知乎回答型内容和专栏型内容完全拆成两套模板

## 12. 下一步顺序

知乎下一步建议按这个顺序继续收敛：

1. 先做品牌 / 型号知识结构
2. 再做更像搜索专栏的知乎长文样稿回归
3. 再拆知乎“回答模板”和“专栏模板”
4. 最后再进入其他平台的一平台一策略收敛
